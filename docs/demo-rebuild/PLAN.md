# 可试玩 Demo 重构计划

## 目标

把当前“功能骨架验证”重构成可试玩暗黑装备放置 Demo。视觉和交互参考用户提供的移动端竖屏暗黑装备放置概念图，但先以 Unity UI Toolkit 原生实现，不依赖外部 UI 包。

视觉基准：

- `docs/demo-rebuild/references/mobile-dark-idle-ui-concept.png`
- `docs/demo-rebuild/references/dark-idle-rpg-ui-target.png`

## Demo 判定标准

1. 打开第一屏像游戏主界面，而不是工程调试面板。
2. 5 分钟内能看懂并体验：自动战斗 -> 掉装 -> 对比 -> 穿戴/强化 -> 推层 -> Boss -> 奖励。
3. 装备、背包、详情、对比、工坊、转生都在一个稳定主界面里，有清晰优先级。
4. 关键反馈可见：战斗推进、Boss 压力、掉落品质、装备评分变化、材料变化、战力变化。
5. 每轮开发必须更新 `docs/demo-rebuild/LOG.md`。

## 执行规则

每一轮开发都必须记录：

- 本轮目标：本次具体要解决什么。
- 完成内容：改了哪些用户可见体验。
- 验证结果：跑了哪些命令，哪些通过，哪些没跑。
- 遗留问题：明确下一轮要补什么。
- 提交信息：验证完成后记录 commit hash。
