# 装备驱动放置游戏 — 设计文档

- **日期**：2026-06-24
- **状态**：已批准，待生成实现计划
- **题材**：西方奇幻地下城
- **类型**：装备驱动放置游戏（联网）

---

## 一、项目画像

| 维度 | 决策 |
|---|---|
| 客户端 | Unity (C#)，2D 像素，导出 Web/Android/iOS/桌面 |
| 服务端 | Go + gorilla/websocket，半权威，JSON 通讯 |
| 部署 | 本地开发联调（Unity 客户端 + 本地 Go 服务器） |
| 长线机制 | 转生重置（MVP 含简化版：魂点 + 3~5 个永久天赋节点） |
| 装备深度 | 高深度硬核词缀（暗黑/POE 式前缀+后缀，60+ 词缀池一步到位） |
| MVP 范围 | 掉落 + 词缀 + 8 槽穿戴 + 分解/合成/重铸 + 强化 + 简化转生 + 离线收益 + 无限层数 |
| 战斗呈现 | 2D 像素，实时简化动画（站桩攻击 + 飘字） |
| 美术 | 免费 CC0/CC-BY 像素素材起步（LPC、itch.io 免费包） |
| 存档 | 服务端权威存档（客户端不持久化关键数据） |

## 二、核心循环

```
离线/在线自动战斗 → 击杀怪物 → 掉落装备
        │
   ┌────┼────────────┬──────────────┐
   ▼    ▼            ▼              ▼
对比穿戴(8槽)   分解得材料     强化(+1~+10)   合成/重铸(定向词缀)
   │    │            │              │
   └────┴─────战力提升──────────────┘
                │
         推进更深层数 → 更好掉落 → 循环放大
                │
       达瓶颈 → 转生(重置进度换魂点永久天赋) → 更快开局
```

**单局心流**：掉装备 → 对比词缀决定换不换 → 多余的分解 → 材料合成/重铸出定向词缀 → 强化主力装备 → 战力涨 → 打更深层 → 掉更好的装备。

## 三、装备系统（灵魂模块）

### 3.1 稀有度与词缀结构

| 稀有度 | 颜色 | 前缀数 | 后缀数 | 特殊 |
|---|---|---|---|---|
| 普通 | 白 | 0 | 0 | — |
| 魔法 | 蓝 | 1 | 1 | — |
| 稀有 | 黄 | 2 | 2 | — |
| 传奇 | 橙 | 1 固定 | 2 随机 | 1 个传奇特效（固定） |
| 神器 | 红 | 2 | 2 | 1 个特殊机制词缀 + 高数值档 |

### 3.2 词缀池（60+，分 Tier 1~5）

每个词缀按 Tier 1~5 派生 5 档数值，从对应档位范围随机取值。

- **基础属性**（6）：力量 / 敏捷 / 智力 / 体力 / 生命 / 护甲
- **衍生属性**（9）：暴击率 / 暴击伤害 / 攻速 / 吸血 / 火伤 / 冰伤 / 雷伤 / 命中 / 闪避
- **经济/特殊**（8+）：掉落率 / 经验加成 / 击杀回血 / 移动速度 / 冷却缩减 / 反伤 / 护盾 / 资源获取

合计 23 个词缀类型 × 5 Tier = **115 个词缀条目**（满足 60+ 要求）。经济/特殊类词缀是后期转生流的核心钩子，提前埋点。

### 3.3 装备槽位（8）

武器 / 头盔 / 护甲 / 手套 / 靴子 / 戒指 ×2 / 项链

### 3.4 装备交互

- **对比穿戴**：捡到装备一键与当前槽位对比，高亮属性差异
- **分解**：任意装备分解为基础材料 + 词缀材料（按词缀 Tier）
- **合成**：材料按配方合成指定基底装备（定向 farming）
- **重铸**：消耗词缀材料重随一件装备的词缀（保留稀有度）
- **强化**：+1~+10，每级提升基础属性 %，有成功率 + 保底（+7 以上失败不掉级）

## 四、战斗与关卡

- **战斗模型**：实时自动战斗，2D 像素站桩攻击 + 飘字伤害，核心是数值结算，不依赖操作
- **关卡**：无限层数地下城，每层怪物强度按公式递增，每 5 层一个 Boss（掉落池更优）
- **战力公式**：`战力 = 攻击 × 攻速 × (1 + 暴击期望) × 生存系数`，用于怪物强度匹配与离线结算
- **怪物强度**：随层数指数式增长，公式待阶段 7 平衡调参时定稿（MVP 先用线性 + 周期性 Boss 峰值）

---

## 五、转生（简化版 MVP）

- **转生条件**：达到层数门槛（MVP 定为第 10 层通关）可转生
- **重置**：当前层数 / 装备 / 材料 / 强化等级
- **保留**：魂点（按历史最高层数计算，公式：`魂点 = floor(历史最高层数 / 5)`）
- **天赋节点**（3~5 个，MVP 选 4 个）：
  1. 全局伤害 +5%/级（最多 10 级）
  2. 初始装备品质 +1 档/级（最多 3 级）
  3. 掉落率 +3%/级（最多 10 级）
  4. 离线收益 +10%/级（最多 5 级）
- 转生增益让下一轮推进更快 → 形成长线动力

---

## 六、离线机制（服务端权威）

- 关闭后角色持续自动战斗，服务端结算：`离线收益 = 当前 DPS × 离线时长 × 0.5 × 掉落表`
- **上限 8 小时**（防无限挂机，鼓励每日回归）
- 离线期间产出的装备进入"待整理"背包，需手动筛选（保留"刷"的仪式感）
- 离线结算纯逻辑放在服务端 `internal/offline`，可单测覆盖

## 七、技术架构

### 7.1 通讯协议

- **传输**：WebSocket 长连接，JSON 消息
- **半权威边界**：
  - 服务端管：存档 / 离线结算 / 掉落生成 / 强化结果 / 转生 / 合成重铸
  - 客户端管：战斗表现插值 / 飘字 / UI 交互 / 本地状态缓存
- **消息分类**：
  - `req/resp`：穿戴 / 卸下 / 分解 / 合成 / 重铸 / 强化 / 转生 / 登录
  - `push`：实时掉落 / 层数推进 / 战斗事件
  - `sync`：登录全量同步（玩家状态 + 背包 + 穿戴 + 层数 + 魂点天赋）
- **消息格式**：`{ "t": "<type>", "id": "<reqId>", "data": {...} }`

### 7.2 服务端目录（Go）

```
server/
├─ cmd/server/main.go          # 入口
├─ internal/
│   ├─ model/                  # 玩家/装备/词缀/背包 领域模型
│   ├─ loot/                   # 装备生成（RNG + 词缀抽取）
│   ├─ combat/                 # 战力计算 + 战斗结算
│   ├─ inventory/              # 背包/穿戴
│   ├─ crafting/               # 合成/重铸/分解
│   ├─ upgrade/                # 强化
│   ├─ reincarnation/          # 转生
│   ├─ offline/                # 离线结算
│   ├─ save/                   # 存档序列化（JSON 文件起步，预留 SQLite）
│   └─ ws/                     # WebSocket handler + 消息路由
├─ data/                       # 静态数据：词缀池/怪物表/配方（JSON）
└─ go.mod
```

### 7.3 客户端目录（Unity / C#）

```
client/                          # Unity 项目根（Assets 内组织如下）
└─ Assets/
   ├─ Scripts/
   │   ├─ Data/                  # 镜像服务端静态数据（词缀/怪物/配方）用于表现
   │   ├─ Net/                   # WebSocket 客户端 + 消息收发（NativeWebSocket 库）
   │   ├─ State/                 # 本地缓存的服务端状态
   │   ├─ Systems/               # 客户端侧战斗插值/飘字/UI 数据桥接
   │   └─ UI/                    # 战斗场景/背包/对比/合成/强化/转生面板
   ├─ Scenes/                    # Main.unity 等
   └─ Art/                       # 像素素材（Asset Store / CC0）
```

### 7.4 设计原则

- 服务端 `internal/` 各包纯逻辑、可单测，不依赖 WebSocket
- 客户端 `Systems/` 只读写 `State/`，`UI/` 只读 `State/` 并发请求
- 静态数据（词缀池 / 怪物表 / 配方）服务端为权威源，客户端启动时同步或随包附带（需校验版本号）

## 八、实现阶段（可验证分步）

### 阶段 0：脚手架与协议
- 初始化 Go 模块、Unity 项目
- 定义 WebSocket 消息协议（JSON schema）
- 打通客户端↔服务端连接 + 登录（临时账号）+ 全量同步
- **验证**：客户端连上服务端，收到空状态同步

### 阶段 1：装备生成核心（服务端，可单测）
- 词缀池数据（115 条目）、装备基底、稀有度
- `loot` 包：RNG + 词缀抽取 + 数值生成
- `combat` 包：战力计算
- **验证**：Go 单测覆盖词缀生成分布、战力公式

### 阶段 2：战斗与掉落循环（服务端权威 + 客户端表现）
- 服务端：怪物表、层数推进、自动战斗结算、掉落
- 客户端：2D 像素战斗场景、飘字、层数 UI
- 消息：登录同步 + 实时掉落 push
- **验证**：客户端看到角色自动打怪、掉装备进背包

### 阶段 3：背包与穿戴（装备交互第一批）
- 服务端 `inventory`：8 槽穿戴、对比、卸下
- 客户端：背包 UI、装备对比面板、一键穿戴
- **验证**：穿戴后战力实时变化，对比高亮正确

### 阶段 4：分解/合成/重铸/强化（装备交互第二批）
- 服务端 `crafting` + `upgrade`：材料、配方、重铸 RNG、强化成功率 + 保底
- 客户端：四个工坊面板
- **验证**：完整装备养成闭环可玩

### 阶段 5：离线结算（服务端）
- `offline` 包：登录时计算上次离线时长收益、装备入待整理背包
- 8 小时上限
- **验证**：关闭再开，正确结算离线掉落

### 阶段 6：简化转生（服务端 + 客户端）
- `reincarnation` 包：转生条件、魂点计算、4 个天赋节点
- 客户端：转生面板、天赋树
- **验证**：转生后重置进度、魂点天赋生效、推进更快

### 阶段 7：打磨与平衡
- 数值平衡调参（词缀档位、怪物强度曲线、强化/转生节奏）
- 素材整合与 UI 统一
- 多端导出测试（Web 优先）

## 九、风险与对策

| 风险 | 对策 |
|---|---|
| 2D 像素素材风格割裂 | 优先 Unity Asset Store 免费像素包 + itch.io CC0 包，选风格统一系列；后期替换 |
| 60+ 词缀数值平衡难 | 阶段 7 专门做平衡；词缀数据外置 JSON 便于热调 |
| 离线结算 bug | 服务端纯逻辑单测覆盖；先用小时长验证再放开 |
| 前后端双端开发量大 | 严格分阶段，每阶段独立可玩可测；服务端先行，客户端跟随 |
| 半权威下客户端作弊 | 关键结果（掉落/强化/转生/存档）全服务端算，客户端只表现 |
